Known domains

Operator-curated list shown in the validator page's domain dropdown alongside each user's own history and any domains with per-domain DKIM config. Comma-, space-, or newline-separated. Domains are normalised (lowercased, trailing dot stripped).

Probe data lists

Advanced. One entry per line. Labelled entries use value|label (the label is shown in reports). Leave a list empty to fall back to the built-in defaults.

Per-domain DKIM selectors

Maintenance

DNS, SMTP and HTTP probe results are cached in memory and on disk to speed up validations.

⚠ Clearing wipes every cached probe (memory and disk). Validations will re-fetch everything and run noticeably slower until the caches warm up again. Use only to force fresh data or reclaim space.
